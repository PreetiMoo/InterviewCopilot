Privacy Policy for InterviewCopilot

Effective date: April 19, 2026

This policy describes how InterviewCopilot handles information based on the current source code of the extension and backend service.

Introduction

InterviewCopilot is a Chrome extension with an optional backend API. It helps users practice interview responses by extracting job-post details from supported job pages and generating interview questions/feedback using Google Gemini.

This policy is intentionally specific to the implemented behavior in the codebase and does not claim data practices that are not present.

Information We Collect

Information extracted from job pages (extension)

  • Job title
  • Company name
  • Job description text
  • Review snippets/culture snippets when available on the page
  • Page URL and scrape timestamp

This extraction runs only on matching LinkedIn/Glassdoor job URLs declared in the extension manifest.

User-provided content

  • Manual company, role title, and job description entered by the user when auto-detection is incomplete
  • Written interview answers entered in the side panel
  • Optional voice dictation content (transcribed text only) while voice mode is active
  • User-selected backend URL in settings

Information sent to backend/API

  • POST /api/generate-questions: jobTitle, company, and cultureSignals
  • POST /api/get-feedback: sessionId, question, answer, jobTitle, and company
  • GET /api/sessions: no request body; returns recent session summaries

Information not collected in current code

  • No account registration or login
  • No name, email, phone number, or government identifiers
  • No payment data
  • No cookies created by this extension code

How We Use Information

  • Populate job context for interview practice (company/role/description/culture signals)
  • Generate interview questions tailored to the role/company context
  • Generate feedback and score for a user’s submitted answer
  • Show history of previous sessions in the side panel
  • Support optional voice dictation in the mock interview screen
  • Store extension settings (for example, backend URL) and current page data locally for UX continuity

Permissions Justification

Permission Why It Is Needed
sidePanel Required to open and run the InterviewCopilot side panel UI when the user clicks the extension popup action.
storage Required to store extension data such as scraped job context, backend URL, and transient voice-update values used by side panel dictation flow.
scripting Required to inject the dictation helper function into the active tab during voice mode and stop that injected listener when voice is turned off.
activeTab Required to run injected dictation code in the current active tab after a user action, without requesting broad host access to all websites.

Host permissions and scope

  • https://www.linkedin.com/*
  • https://www.glassdoor.com/*
  • https://www.glassdoor.co.in/*
  • http://localhost/*
  • http://127.0.0.1/*

LinkedIn/Glassdoor permissions are used for job-page content extraction on supported pages. Localhost host permissions are used for extension-to-backend API requests in local/development deployments.

Data Storage and Security

Client-side (Chrome extension)

  • Uses chrome.storage.local for local extension state (job page data, backend URL, temporary voice update payloads).
  • No use of localStorage, sessionStorage, or IndexedDB in current code.

Server-side

  • Uses MongoDB via Mongoose to store interview sessions.
  • Stored session fields include company, job title, culture signals, generated questions, submitted answers, feedback, score, and creation timestamp.

Security controls such as TLS termination, database access policy, and infrastructure hardening depend on how you deploy the backend. Those controls are not defined by this repository alone.

Third-Party Services

  • Google Gemini API: The backend sends prompt content containing job context and answer text to Google Gemini to generate questions and feedback.
  • MongoDB: Session data is stored in a MongoDB database defined by your MONGODB_URI.

No analytics SDK (for example Google Analytics, Mixpanel, Segment, or Sentry) is present in the current extension/server code.

User Rights

  • You can edit or stop sending new data by not using generation/feedback actions and by disabling voice mode.
  • You can clear extension-local data through Chrome extension storage controls (or by uninstalling the extension).
  • Server-stored sessions are managed by the backend deployment owner; this codebase currently provides read-listing of recent sessions but no dedicated delete endpoint.

Data Retention

  • Extension local data remains until overwritten, manually cleared, or the extension is removed.
  • MongoDB session records do not have an automatic expiration (TTL) in the current schema and are retained until manually deleted by the operator.
  • Voice updates in extension storage are temporary operational values and are removed when voice stops in normal flow.

Changes to This Policy

If app behavior changes (for example new permissions, new third-party services, or additional data fields), this policy should be updated before release. The effective date at the top should be revised whenever changes are made.
